- 页面逻辑文件 (`index.js`) 统一通过 `require` 引入 `utils/` 下的功能模块，避免在页面内直接编写底层网络或认证逻辑。
- 网络请求统一使用 `utils/request.js` 封装的 `request` 方法，该方法自动处理 Base URL 拼接、Token 注入及标准响应码（code === 0）校验。
- 用户认证状态强依赖本地存储 (`wx.getStorageSync('token')`)，并在 `app.js` 中维护全局同步，页面加载时优先检查全局或本地登录态。
- API 环境配置支持动态切换，通过 `wx.setStorageSync('apiEnv')` 或 `customBaseUrl` 覆盖默认的生产/本地地址，便于调试。
