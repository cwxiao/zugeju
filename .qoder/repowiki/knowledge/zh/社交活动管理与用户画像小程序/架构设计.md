- **入口与全局状态**：`app.js` 作为应用入口，管理全局用户登录状态（token/user）及品牌配置，通过 `onLaunch` 同步本地存储的登录态。
- **页面路由结构**：`app.json` 定义了8个核心页面路由，涵盖首页、活动创建/详情、活动中心、账单、人格画像等模块，采用标准的微信小程序 Page 结构（js/json/wxml/wxss）。
- **工具层封装**：`utils/` 目录提供核心业务能力：
  - `request.js`：封装 `wx.request`，支持多环境 API 地址切换（本地/生产/自定义）、自动注入 Bearer Token 及 401/403 自动登出处理。
  - `auth.js`：实现微信一键登录流程，结合 `wx.login` 与后端接口完成用户身份置换与本地持久化。
  - `activity-insights.js`：内置静态种子数据与算法逻辑，用于生成用户活动画像、社交标签及动物人格报告。
  - `subscribe.js`：处理微信订阅消息权限请求。
- **数据流模式**：页面通过 `require` 引入工具函数，直接调用 `wx` API 或封装后的 request/auth 方法，状态变更主要通过 `setData` 驱动视图更新，依赖 `wx.getStorageSync/setStorageSync` 进行跨页面数据共享。
