# 跨小程序联动侧边抽屉组件规范

> 版本：1.0  
> 更新：2026-06-10  
> 适用：所有自有微信小程序/小游戏

---

## 1. 设计理念

在所有自有小程序/小游戏的首页右侧，放置一个统一的竖向标签入口，点击后从右侧滑出抽屉面板，展示其他自有产品的推荐卡片，实现产品矩阵内的交叉引流。

**核心原则**：
- **不侵入主流程**：侧边标签固定在屏幕右侧，不影响任何页面主体内容和交互
- **统一视觉风格**：所有产品使用相同的抽屉样式，形成品牌辨识度
- **一键跳转**：使用 `wx.navigateToMiniProgram` 直接跳转，无需扫码
- **轻量可配置**：通过数据配置即可增减推荐项，无需改 UI 代码

## 2. 视觉规格

### 2.1 侧边标签（收起态）

| 属性 | 值 |
|------|------|
| 位置 | `position: fixed; right: 0; top: 50%; transform: translateY(-50%)` |
| 方向 | 竖排文字，`writing-mode: vertical-rl` |
| 尺寸 | `padding: 18rpx 10rpx`，自适应内容高度 |
| 圆角 | `border-radius: 16rpx 0 0 16rpx`（左上左下圆角，右侧贴屏幕） |
| 背景 | 橙色渐变 `linear-gradient(180deg, rgba(240,160,48,0.92), rgba(232,120,40,0.92))` |
| 文字 | 白色、700 字重、20rpx、竖排、2rpx 字间距 |
| 阴影 | `box-shadow: -4rpx 4rpx 20rpx rgba(232,120,40,0.35)` |
| 动画 | 呼吸脉冲 `drawer-tab-pulse`，3s 循环，阴影从 0.35 → 0.55 → 0.35 |
| 层级 | `z-index: 999` |

### 2.2 遮罩层

| 属性 | 值 |
|------|------|
| 位置 | `position: fixed; inset: 0` |
| 背景 | `rgba(0,0,0,0.4)` |
| 层级 | `z-index: 1000` |
| 动画 | 0.2s 淡入 |

### 2.3 抽屉面板（展开态）

| 属性 | 值 |
|------|------|
| 位置 | `position: fixed; right: 0; top: 50%; transform: translateY(-50%)` |
| 宽度 | `320rpx` |
| 背景 | `linear-gradient(165deg, rgba(255,250,243,0.98), rgba(255,235,210,0.96))` |
| 圆角 | `border-radius: 28rpx 0 0 28rpx` |
| 阴影 | `box-shadow: -8rpx 8rpx 36rpx rgba(0,0,0,0.15)` |
| 层级 | `z-index: 1001` |
| 动画 | `transition: right 0.3s cubic-bezier(0.4, 0, 0.2, 1)` |
| 收起位置 | `right: -400rpx` |
| 展开位置 | `right: 0` |

## 3. 数据配置

### 3.1 推荐项配置结构

每个推荐项的数据结构：

```javascript
{
  tabText: '旋天阵',           // 侧边标签显示的文字（竖排，建议2-4字）
  title: '🎮 换个玩法？',       // 面板标题
  coverImage: '/img/xuantianzhen.jpg',  // 封面图路径
  name: '旋天阵',              // 产品名称
  desc: '策略对弈，指尖排兵布阵', // 一句话描述
  buttonText: '立即去玩 ▸',    // 按钮文字
  appId: 'wx0be195871454d924',  // 目标小程序 AppID
  path: '',                     // 跳转路径（空字符串=首页）
  envVersion: 'release'         // release / trial / develop
}
```

## 4. 代码模板

### 4.1 WXML 模板

```xml
<!-- 跨产品联动侧边抽屉 -->
<view class="game-drawer-tab" bindtap="toggleGameDrawer">
  <text class="game-drawer-tab-text">{{drawerItem.tabText}}</text>
</view>

<view wx:if="{{gameDrawerOpen}}" class="game-drawer-mask" bindtap="closeGameDrawer"></view>

<view class="game-drawer-panel {{gameDrawerOpen ? 'game-drawer-panel-open' : ''}}">
  <view class="game-drawer-header">
    <text class="game-drawer-title">{{drawerItem.title}}</text>
    <view class="game-drawer-close" bindtap="closeGameDrawer">✕</view>
  </view>
  <view class="game-drawer-body">
    <image class="game-drawer-cover" src="{{drawerItem.coverImage}}" mode="aspectFill" />
    <text class="game-drawer-name">{{drawerItem.name}}</text>
    <text class="game-drawer-desc">{{drawerItem.desc}}</text>
    <view class="game-drawer-play-btn" bindtap="goToDrawerTarget">
      <text class="game-drawer-play-text">{{drawerItem.buttonText}}</text>
    </view>
  </view>
</view>
```

### 4.2 JS 逻辑

```javascript
// 在 Page data 中添加
data: {
  gameDrawerOpen: false,
  drawerItem: GAME_DRAWER_ITEMS[0] // 或根据当前产品选择不同的推荐项
},

// 方法
toggleGameDrawer() {
  this.setData({ gameDrawerOpen: !this.data.gameDrawerOpen })
},

closeGameDrawer() {
  this.setData({ gameDrawerOpen: false })
},

goToDrawerTarget() {
  const item = this.data.drawerItem
  wx.navigateToMiniProgram({
    appId: item.appId,
    path: item.path || '',
    envVersion: item.envVersion || 'release',
    success: () => {
      this.setData({ gameDrawerOpen: false })
    },
    fail: (err) => {
      console.error('跳转失败', err)
      wx.showToast({ title: '跳转失败，请稍后重试', icon: 'none' })
    }
  })
}
```

## 5. 接入流程

### 5.1 新产品接入步骤

1. **准备封面图**：将产品封面/二维码截图放到 `/img/` 目录，建议 260×260rpx 以上，格式 JPG/PNG
2. **获取 AppID**：从微信公众后台 → 开发管理 → 开发设置 获取
3. **配置数据**：在页面的 JS 文件中配置 `drawerItem` 对象
4. **复制模板**：将 WXML、WXSS、JS 代码复制到目标页面
5. **联调测试**：在开发工具中测试跳转是否正常（`envVersion` 先用 `develop`）

### 5.2 注意事项

- `wx.navigateToMiniProgram` 需要在微信公众后台配置**跳转列表**（设置 → 关联设置 → 跳转小程序），否则无法跳转
- 封面图建议使用产品的 Logo 或特色画面，而非纯二维码（跳转已由 API 完成，无需扫码）
- `envVersion` 在开发阶段用 `develop`，体验版用 `trial`，上线后改为 `release`
- 侧边标签文字建议 **2-4 字**，竖排显示效果最佳
- 面板标题可用 Emoji 增加趣味性，如 `🎮 换个玩法？`、`🎯 再来一局？`

## 7. 多产品扩展方案

当推荐产品超过 1 个时，建议将面板改为**纵向滚动列表**：

```xml
<view class="game-drawer-panel {{gameDrawerOpen ? 'game-drawer-panel-open' : ''}}">
  <view class="game-drawer-header">
    <text class="game-drawer-title">🎮 更多好玩</text>
    <view class="game-drawer-close" bindtap="closeGameDrawer">✕</view>
  </view>
  <scroll-view scroll-y class="game-drawer-scroll">
    <view class="game-drawer-item" wx:for="{{drawerItems}}" wx:key="appId">
      <image class="game-drawer-cover-sm" src="{{item.coverImage}}" mode="aspectFill" />
      <view class="game-drawer-info">
        <text class="game-drawer-name-sm">{{item.name}}</text>
        <text class="game-drawer-desc-sm">{{item.desc}}</text>
      </view>
      <view class="game-drawer-play-btn-sm" bindtap="goToDrawerTarget" data-app-id="{{item.appId}}" data-path="{{item.path}}" data-env="{{item.envVersion}}">
        <text class="game-drawer-play-text-sm">去玩</text>
      </view>
    </view>
  </scroll-view>
</view>
```

对应扩展 CSS 和 JS 逻辑按需补充，核心交互模式不变。

## 8. 品牌色适配

当前默认橙色渐变适用于暖色系产品。如果目标产品风格差异较大，可按以下规则适配：

| 产品色调 | 标签渐变 | 按钮渐变 |
|---------|---------|---------|
| 暖色（默认） | `rgba(240,160,48,0.92) → rgba(232,120,40,0.92)` | `#f0a030 → #e87828` |
| 冷色/科技感 | `rgba(66,165,245,0.92) → rgba(30,136,229,0.92)` | `#42a5f5 → #1e88e5` |
| 绿色/自然 | `rgba(102,187,106,0.92) → rgba(67,160,71,0.92)` | `#66bb6a → #43a047` |
| 紫色/游戏 | `rgba(171,71,188,0.92) → rgba(142,36,170,0.92)` | `#ab46bc → #8e24aa` |

适配时替换标签、按钮的渐变色和阴影色即可，面板背景和文字色保持不变。
