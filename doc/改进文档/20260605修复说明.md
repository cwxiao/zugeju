## 问题1：首页"开始使用"按钮无响应

### 📋 问题描述

**现象**：未登录状态下，首页显示"功能介绍"卡片，点击"开始使用"按钮无任何反应

**影响**：用户无法触发授权登录流程，导致无法使用需要登录的功能

**严重程度**：🔴 高（阻塞核心流程）

---

### 🔍 问题分析

#### 1. WXML 绑定检查

**文件**：`frontend/pages/home/index.wxml`

```xml
<button class="btn-primary" bindtap="showAuth">开始使用</button>
```

✅ **WXML 绑定正确**：`bindtap="showAuth"` 绑定正常

#### 2. JS 方法检查

**文件**：`frontend/pages/home/index.js`

**问题发现**：JS 文件中**没有 `showAuth()` 方法**！

```javascript
// ❌ 原有代码（第160-180行附近）
goPersonality() {
  if (!getApp().hasLoginState()) {
    this.setData({ authVisible: true })
    return
  }
  wx.navigateTo({
    url: '/pages/personality/index'
  })
},
// ↑ 这里直接就到了 goPersonality()，没有 showAuth() 方法
```

**根本原因**：
- WXML 中绑定了 `showAuth` 事件
- 但 JS 中未定义 `showAuth()` 方法
- 点击按钮时，微信小程序找不到对应的事件处理函数，导致无响应

---

### ✅ 修复方案

#### 修复代码位置

**文件**：`frontend/pages/home/index.js`

**插入位置**：在第166行（`goPersonality()` 方法前）

```javascript
  showAuth() {
    this.setData({ authVisible: true })
  },

  goPersonality() {
    // ... 原有代码
  }
```

#### 核心逻辑说明

**`showAuth()` 方法的作用**：
1. 设置 `authVisible: true`
2. 触发授权弹窗显示
3. 用户可以选择"允许"或"拒绝"授权

**授权弹窗的显示逻辑**：
```javascript
// 在 WXML 中
<view wx:if="{{authVisible}}" class="auth-modal">
  <!-- 授权弹窗内容 -->
  <button open-type="chooseAvatar" bindchooseavatar="onChooseAvatar">授权头像</button>
  <input type="nickname" bindinput="onInputNickname" />
  <button bindtap="confirmAuth">确认</button>
  <button bindtap="closeAuth">先随便看看</button>
</view>
```

**为什么这样设计**：
- 符合微信审核规范：用户先浏览功能，再自行选择授权
- 授权弹窗可关闭：用户可以选择"先随便看看"
- 授权后可以正常使用所有功能

## 问题2：成员头像不显示

### 📋 问题描述

**现象**：活动详情页的参与成员列表中，头像图片不显示，只显示默认占位符

**影响**：用户体验差，无法直观看到哪些好友参与了活动

**严重程度**：🟡 中（影响用户体验）

---

### 🔍 问题分析

#### 1. 数据流检查

**后端返回数据**：✅ 正常

**文件**：`backend/src/main/java/com/playminipro/activity/mapper/ActivityMemberMapper.java`

```java
// 第60-72行：SQL 查询包含 avatar_url 字段
@Select({
  "<script>",
  "SELECT u.id, u.nickname, u.avatar_url as \"avatarUrl\", ...",
  "FROM activity_member am",
  "JOIN \"user\" u ON am.user_id = u.id",
  "WHERE am.activity_id = #{activityId}",
  "</script>"
})
List<ActivityMemberResponse> findByActivityId(@Param("activityId") Long activityId);
```

✅ **后端数据正常**：`avatarUrl` 字段已正确查询并返回

#### 2. 前端数据绑定检查

**文件**：`frontend/pages/detail/index.wxml`

```xml
<view class="avatar-pill avatar-image-pill">
  <image wx:if="{{item.avatarUrl}}" 
         class="avatar-pill-image" 
         src="{{item.avatarUrl}}" 
         mode="aspectFill" />
  <text wx:else>{{item.shortName}}</text>
</view>
```

✅ **数据绑定正确**：`wx:if="{{item.avatarUrl}}"` 判断正常

#### 3. 根本原因分析

**问题**：微信头像 URL **有时效性**，且可能存在以下问题：

1. **URL 过期**：微信返回的头像 URL 不是永久有效的
2. **网络问题**：HTTPS 请求失败
3. **图片加载失败**：URL 返回 404 或 403

**原生行为**：
- 当 `<image>` 加载失败时，**不会自动触发任何事件**
- 图片区域显示空白或破损图标
- `wx:if="{{item.avatarUrl}}"` 只判断 URL 是否存在，不判断是否能加载成功

---

### ✅ 修复方案

#### 核心思路

**添加错误处理**：
1. 给 `<image>` 标签添加 `binderror` 事件
2. 当图片加载失败时，触发错误处理函数
3. 在数据中标记 `avatarError: true`
4. WXML 根据 `avatarError` 状态显示备用内容（昵称首字符）

#### 修复代码位置

##### 1. WXML 添加错误监听

**文件**：`frontend/pages/detail/index.wxml`

**修改前**：
```xml
<image wx:if="{{item.avatarUrl}}" 
       class="avatar-pill-image" 
       src="{{item.avatarUrl}}" 
       mode="aspectFill" />
```

**修改后**：
```xml
<image wx:if="{{item.avatarUrl && !item.avatarError}}" 
       class="avatar-pill-image" 
       src="{{item.avatarUrl}}" 
       mode="aspectFill" 
       binderror="onAvatarError" 
       data-index="{{index}}" />
<text wx:else>{{item.shortName}}</text>
```

**关键改动**：
1. 添加 `binderror="onAvatarError"`：监听图片加载失败事件
2. 添加 `data-index="{{index}}"`：传递当前成员索引，方便定位
3. 修改 `wx:if` 条件：增加 `!item.avatarError` 判断

##### 2. JS 添加错误处理函数

**文件**：`frontend/pages/detail/index.js`

**插入位置**：在第302行（`cancelActivity()` 方法前）

```javascript
onAvatarError(event) {
  const index = event.currentTarget.dataset.index
  const members = this.data.detail.members
  if (members && members[index]) {
    this.setData({
      [`detail.members[${index}].avatarError`]: true
    })
  }
},

cancelActivity() {
  // ... 原有代码
}
```

**核心逻辑说明**：

1. **获取索引**：
   ```javascript
   const index = event.currentTarget.dataset.index
   ```
   - 从 `data-index="{{index}}"` 中获取当前成员的索引

2. **检查数据有效性**：
   ```javascript
   const members = this.data.detail.members
   if (members && members[index]) {
     // ...
   }
   ```
   - 防止数组越界或数据不存在的情况

3. **设置错误标记**：
   ```javascript
   this.setData({
     [`detail.members[${index}].avatarError`]: true
   })
   ```
   - 使用**路径语法** `detail.members[${index}].avatarError`
   - 只标记当前失败的头像，不影响其他成员

4. **WXML 自动更新**：
   - 当 `avatarError` 变为 `true` 后
   - `wx:if="{{item.avatarUrl && !item.avatarError}}"` 判断失败
   - 显示 `<text wx:else>{{item.shortName}}</text>`（昵称首字符）

---

### 🎨 优化效果

**修复前**：
- 头像加载失败 → 显示空白
- 用户体验差

**修复后**：
- 头像加载失败 → 自动显示昵称首字符
- 例如：昵称 "张三" → 显示 "张"
- 用户体验友好

## 问题3：接受/婉拒按钮布局异常

### 📋 问题描述

**现象**：
- 未加入活动时，显示"接受邀请"和"婉拒"两个按钮（布局正常）
- 已加入活动后，显示"退出活动"按钮，但**布局奇怪**（按钮被拉伸或居中异常）

**影响**：UI 显示不美观，不符合设计规范

**严重程度**：🟢 低（仅影响视觉效果）

---

### 🔍 问题分析

#### 1. WXML 结构检查

**文件**：`frontend/pages/detail/index.wxml`

**原有代码**：
```xml
<view wx:if="{{detail.canJoin}}" class="invite-action-row">
  <button class="invite-accept-button" bindtap="acceptInvite">接受邀请</button>
  <button class="invite-decline-button" bindtap="declineInvite">婉拒</button>
</view>
<view wx:if="{{detail.currentUserJoined && !detail.isCreator && detail.rawStatus !== 'finished' && detail.rawStatus !== 'cancelled'}}" class="invite-action-row">
  <button class="invite-decline-button" bindtap="quitActivity">退出活动</button>
</view>
```

**问题发现**：
- "接受邀请"和"婉拒"使用 `invite-action-row` 容器（双按钮布局）
- "退出活动"也使用 `invite-action-row` 容器（单按钮布局）
- **但 CSS 中 `.invite-action-row` 设置了 `grid-template-columns: 1fr 1fr`（两列布局）**

#### 2. CSS 样式检查

**文件**：`frontend/pages/detail/index.wxss`

**原有样式**：
```css
.invite-action-row {
  display: grid;
  grid-template-columns: 1fr 1fr;  /* ← 问题在这里！ */
  gap: 16rpx;
  margin-top: 24rpx;
}
```

**布局计算结果**：
- 双按钮（接受 + 婉拒）：
  - 第1列：接受邀请按钮
  - 第2列：婉拒按钮
  - ✅ 布局正常

- 单按钮（退出活动）：
  - 第1列：退出活动按钮
  - 第2列：空白
  - ❌ 按钮被拉伸到第1列的全宽，或者布局奇怪

---

### ✅ 修复方案

#### 核心思路

**分离布局**：
1. 双按钮（接受 + 婉拒）使用 `.invite-action-row`（网格布局）
2. 单按钮（退出活动）使用 `.invite-action-row-single`（弹性布局，居中显示）

#### 修复代码位置

##### 1. WXML 分离容器类

**文件**：`frontend/pages/detail/index.wxml`

**修改前**：
```xml
<view wx:if="{{detail.currentUserJoined && !detail.isCreator && detail.rawStatus !== 'finished' && detail.rawStatus !== 'cancelled'}}" class="invite-action-row">
  <button class="invite-decline-button" bindtap="quitActivity">退出活动</button>
</view>
```

**修改后**：
```xml
<view wx:if="{{detail.currentUserJoined && !detail.isCreator && detail.rawStatus !== 'finished' && detail.rawStatus !== 'cancelled'}}" class="invite-action-row-single">
  <button class="invite-decline-button invite-quit-button" bindtap="quitActivity">退出活动</button>
</view>
```

**关键改动**：
1. 容器类从 `invite-action-row` 改为 `invite-action-row-single`
2. 按钮添加 `invite-quit-button` 类，用于限制宽度

##### 2. WXSS 添加单按钮布局样式

**文件**：`frontend/pages/detail/index.wxss`

**添加样式**：
```css
.invite-action-row {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: 16rpx;
  margin-top: 24rpx;
}

/* ↑ 保留双按钮布局 */

/* ↓ 新增单按钮布局 */
.invite-action-row-single {
  display: flex;
  justify-content: center;
  margin-top: 24rpx;
}

.invite-action-row-single .invite-quit-button {
  width: 60%;
  max-width: 400rpx;
}
```

**核心逻辑说明**：

1. **`.invite-action-row-single`**：
   - `display: flex`：弹性布局
   - `justify-content: center`：水平居中
   - 让按钮在容器中居中显示

2. **`.invite-quit-button`**：
   - `width: 60%`：按钮宽度为容器的 60%
   - `max-width: 400rpx`：最大宽度限制，避免在大屏上过宽
   - 让按钮看起来协调，不会太宽或太窄

---

### 🎨 优化效果

**修复前**：
- "退出活动"按钮使用网格布局（两列）
- 按钮被拉伸或布局奇怪
- UI 不美观

**修复后**：
- "退出活动"按钮使用弹性布局（居中）
- 按钮宽度为容器的 60%，最大 400rpx
- UI 美观，符合设计规范
