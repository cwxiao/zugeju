# AI人格档案（AI Personality Report）

类似：

- 网易云年度总结
- 支付宝年度账单
- QQ年度报告
- Spotify Wrapped

用户愿意分享。

如果只是：

```text
年龄：25
兴趣：游戏
标签：社牛
```

没人会分享。

但如果是：

```text
你的称号：

深夜整活王

过去30天：

你发起了17场活动

其中12场发生在晚上10点以后

AI评价：

“别人下班回家，你下班开局”
```

用户就会截图发朋友圈。

# 来整 AI人格档案 PRD V1.0

------

# 一、产品目标

## 产品定位

通过分析用户在「来整」中的行为数据，

生成：

- 兴趣画像
- 社交画像
- 活跃画像
- AI锐评
- 荣誉成就

形成可分享的 AI 人格档案。

# 四、整活指数

满分100

------

## 计算公式

```text
发起活动 × 5

参与活动 × 3

邀请好友 × 2

连续活跃 × 1
```

例如：

```text
发起 10次

参与 20次

邀请 30次

活跃 15天
```

得分：

```text
50+60+60+15

=185

映射为95分
```

------

## 展示

```text
整活指数

95分

击败98%的用户
```

# 九、AI锐评模块

调用大模型

输入：

```json
{
  "nickname":"张三",
  "launchCount":12,
  "joinCount":35,
  "nightActivityPercent":70,
  "favorite":"电竞",
  "inviteCount":52
}
```

Prompt：

```text
你是一个幽默风趣的社交分析师。

根据用户活动数据，

生成：

1句称号

3句幽默评价

不要攻击用户

字数100字以内
```

返回：

```text
称号：

深夜整活王

锐评：

过去30天，

你发起了12场活动，

其中70%发生在深夜。

别人晚上刷短视频，

你晚上摇人开黑。

你的通讯录里，

一定有几个随叫随到的兄弟。
```


------

## AI成本优化

不要实时调用。

推荐：

```text
每日凌晨批量生成

写库缓存
```

用户查看直接读取。

这样即使 10 万用户：

```text
一天只调用10万次
```

成本非常低。
